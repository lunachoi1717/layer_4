# 무신사 쇼핑몰 포트폴리오 실행 가이드

## 기술 스택
- **Backend**: Spring Boot 3.2.0 + Java 21 + Spring Security + JPA/Hibernate
- **Database**: PostgreSQL 14+
- **Frontend**: Vue.js 3 (Composition API) + Vite + Vue Router 4

---

## 1. 사전 준비 — PostgreSQL 설정

### 1-1. PostgreSQL 서비스 시작
```bash
# Windows (서비스 관리자에서 PostgreSQL 시작)
# 또는 PowerShell 관리자 권한으로:
net start postgresql-x64-14

# macOS
brew services start postgresql

# Linux
sudo systemctl start postgresql
```

### 1-2. DB 생성 및 사용자 설정
```sql
-- psql 접속 (postgres 계정)
psql -U postgres

-- DB 생성
CREATE DATABASE shop;

-- 비밀번호 확인/설정 (application.yml의 password: 1004 와 일치해야 함)
ALTER USER postgres PASSWORD '1004';

-- 종료
\q
```

### 1-3. DDL 실행 순서 (중요!)

**반드시 아래 순서로 실행하세요:**

```bash
# 1번 파일 먼저 실행 (기본 테이블 + 상품 더미데이터 112개)
psql -U postgres -d shop -f musinsa_ddl1.sql

# 2번 파일 실행 (컬럼 추가 + 리뷰/Q&A 테이블 + 더미 회원/리뷰/Q&A 데이터)
psql -U postgres -d shop -f musinsa_ddl2.sql
```

> 💡 IntelliJ 내부 Database 도구를 사용하는 경우:
> - `shop` DB에 연결 후 SQL 파일을 각각 열어 `Run` 버튼 클릭

## 2. 백엔드 실행 (IntelliJ IDEA)

### 2-1. 프로젝트 열기
1. IntelliJ IDEA 실행
2. `File` → `Open` → `shop_musinsa/shop` 폴더 선택
3. Gradle 동기화 완료 대기 (우측 하단 진행바)

### 2-2. application.yml 확인
```
shop_musinsa/shop/src/main/resources/application.yml
```
DB 비밀번호가 PostgreSQL과 일치하는지 확인:
```yaml
datasource:
  url: jdbc:postgresql://localhost:5432/shop
  username: postgres
  password: 1004   ← PostgreSQL postgres 사용자 비밀번호
```

### 2-3. 백엔드 시작
1. `ShopApplication.java` 파일 열기
   - 경로: `src/main/java/com/musinsa/shop/ShopApplication.java`
2. 클래스 옆 ▶ 버튼 클릭 → `Run 'ShopApplication'`
3. 콘솔에 `Started ShopApplication` 메시지 확인
4. 포트: **8080**

### 2-4. 자동 처리 사항 (AdminInitializer)
백엔드 최초 실행 시 자동으로:
- 관리자 계정 생성: `admin@musinsa.com` / `admin1234`
- DB의 평문 비밀번호를 BCrypt로 마이그레이션

## 3. 프론트엔드 실행

### 3-1. Node.js 요구사항
- Node.js 18+ 설치 필요
- [다운로드](https://nodejs.org)

### 3-2. 의존성 설치
```bash
# 터미널에서 pure 폴더로 이동
cd pure

# 패키지 설치
npm install
```

### 3-3. 개발 서버 시작
```bash
npm run dev
```
실행 후 브라우저에서: **http://localhost:5173**

> 💡 Vite 프록시 설정으로 `/v1/*` 요청은 자동으로 `http://localhost:8080`으로 전달됩니다.

## 7. 자주 발생하는 오류

### 포트 충돌
```bash
# 8080 포트 사용 프로세스 종료 (Windows)
netstat -ano | findstr :8080
taskkill /PID {PID번호} /F
```

### DB 연결 실패 (`EOFException`, `Unable to determine Dialect`)
- PostgreSQL 서비스가 실행 중인지 확인
- `shop` DB가 존재하는지 확인
- `application.yml`의 비밀번호 확인

### 이미지 업로드 경로
- 업로드된 이미지는 `shop/uploads/` 폴더에 저장됩니다
- URL: `http://localhost:8080/v1/images/{파일명}`

### CORS 오류
- 백엔드가 실행 중인지 확인 (포트 8080)
- Vite dev server가 실행 중인지 확인 (포트 5173)
